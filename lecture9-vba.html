LECTURE 9: VBA — FORMS, CONTROLS & CAPM

Interface programming with CheckBoxes, OptionButtons, ToggleButton, UserForms, and the CAPM calculator.

1. Form Controls — CheckBox / OptionButton / ToggleButton

These procedures are linked to interface controls placed on the worksheet. They run when the user clicks a specific control — not like normal macros.

🎨 CheckBox Group — Color Toggle

Each checkbox changes its background color when checked.

' CheckBox color toggle
Private Sub CheckBox1_Click() If CheckBox1.Value = True Then CheckBox1.BackColor = vbBlue Else CheckBox1.BackColor = vbWhite End If End Sub Private Sub CheckBox2_Click() If CheckBox2.Value = True Then CheckBox2.BackColor = vbRed Else CheckBox2.BackColor = vbWhite End If End Sub Private Sub CheckBox3_Click() If CheckBox3.Value = True Then CheckBox3.BackColor = vbYellow Else CheckBox3.BackColor = vbWhite End If End Sub

🧠 Core idea

A checkbox is a yes/no control. Its key property is Value:

  • True = checked → color changes
  • False = unchecked → back to white

BackColor controls the background color of the control.

These procedures are purely visual — they don't calculate anything. They show how VBA reacts to user clicks.

🔘 OptionButton Group — Choose an Operation

Each button performs a different calculation on cells E15 and E16, then displays the result in a TextBox.

' OptionButtons — sum / multiply / divide
Private Sub OptionButton1_Click() If OptionButton1.Value = True Then TextBox1.Text = "Result: " & Str(Range("E15").Value + Range("E16").Value) End If End Sub Private Sub OptionButton2_Click() If OptionButton2.Value = True Then TextBox1.Text = "Result: " & Str(Range("E15").Value * Range("E16").Value) End If End Sub Private Sub OptionButton3_Click() If OptionButton3.Value = True Then TextBox1.Text = "Result: " & Str(Range("E15").Value / Range("E16").Value) End If End Sub

🧠 Core idea

OptionButtons are mutually exclusive — only one can be selected at a time.

  • Button 1 → E15 + E16
  • Button 2 → E15 × E16
  • Button 3 → E15 ÷ E16

Str(…) converts the result to text because TextBox1.Text expects a string, not a number.

The result is displayed inside the interface, not written to the worksheet.

🔀 ToggleButton — Load / Clear Lists

Acts like an ON/OFF switch: ON loads data into a ComboBox and ListBox, OFF clears them.

' ToggleButton — load or clear lists
Private Sub ToggleButton1_Click() Dim i As Integer If ToggleButton1.Value = True Then Range("G24").Activate Do While ActiveCell.Value <> "" ComboBox1.AddItem(ActiveCell.Value) ListBox1.AddItem(ActiveCell.Value) ActiveCell.Offset(1, 0).Activate Loop ComboBox1.ListIndex = 0 Else ComboBox1.Clear ListBox1.Clear End If Range("G1").Activate End Sub

🧠 Core idea

When ON: reads a vertical list from G24 downward until an empty cell, adding each value to both ComboBox1 (dropdown) and ListBox1 (visible list).

ComboBox1.ListIndex = 0 → selects the first item by default.

When OFF: both controls are cleared.

One toggle button manages both states — no need for two separate buttons.

📋 Summary

  • CheckBoxes → change appearance (True / False)
  • OptionButtons → select one operation, display result in TextBox
  • ToggleButton → ON loads lists, OFF clears them

2. CAPM — UserForm Calculator

📘 What is CAPM?

Capital Asset Pricing Model — estimates the expected return of an asset.

E(Rᵢ) = Rf + β × (E(Rm) − Rf)

  • Rf = risk-free rate (safe investment)
  • E(Rm) = expected market return
  • β = sensitivity to market movements (systematic risk)
  • E(Rᵢ) = expected return of the asset

Intuition: start with the safe return Rf, then add a risk premium = how much extra the market pays × how risky the asset is.

  • β = 1 → same risk as the market
  • β > 1 → more risky
  • β < 1 → less risky

📗 CommandButton1 — Calculate CAPM

' CAPM calculation from UserForm inputs
Private Sub CommandButton1_Click() rf = UserForm1.ComboBoxRF.Text beta = UserForm1.TextBoxBeta.Text erm = UserForm1.TextBoxRM.Text If VBA.IsNumeric(rf) And VBA.IsNumeric(beta) And VBA.IsNumeric(erm) Then er = rf + (erm - rf) * beta End If UserForm1.TextBoxR.Text = er End Sub

🧠 Breakdown

1. Read inputs from the form

  • ComboBoxRF → risk-free rate (Rf)
  • TextBoxBeta → beta (β)
  • TextBoxRM → expected market return (E(Rm))

2. Validate inputs

VBA.IsNumeric(…) checks that all three values are numbers.

3. Apply the CAPM formula

er = rf + (erm - rf) * beta

→ (erm - rf) = market risk premium
→ × beta = adjusted to the asset's risk
→ + rf = total expected return

4. Display result

The result appears in TextBoxR — directly inside the form.

📕 CommandButton2 — Hide & Clear

' Hide form and clear all fields
Private Sub CommandButton2_Click() UserForm1.Hide UserForm1.ComboBoxRF.Text = "" UserForm1.TextBoxBeta.Text = "" UserForm1.TextBoxRM.Text = "" UserForm1.TextBoxR.Text = "" End Sub

🧠 Breakdown

UserForm1.Hide → closes the form from the screen.

Then all input and output fields are cleared to empty strings.

So the next time the form opens, it starts clean.

💡 Why is this inside a UserForm?

This code is form-event code — not standard module code, not worksheet code.

The buttons belong to the form itself. When the user clicks, the form reacts immediately. That's why the procedures are written as Private Sub CommandButton_Click().

3. UserForm Preparation & ActiveCell Inspector

Two utility macros: one prepares and launches the UserForm, the other inspects the currently selected cell.

📗 Button2_Click — Prepare & Open UserForm

' Prepare ComboBox and show UserForm
Option Explicit Sub Button2_Click() UserForm1.ComboBoxRF.Clear For i = 1 To 8 UserForm1.ComboBoxRF.AddItem(Str(i * 0.5)) Next UserForm1.Show End Sub

🧠 Breakdown

1. Clear old items

UserForm1.ComboBoxRF.Clear

→ prevents duplicates if the macro runs multiple times.

2. Fill with risk-free rate values

For i = 1 To 8 UserForm1.ComboBoxRF.AddItem(Str(i * 0.5)) Next

→ adds 0.5, 1.0, 1.5, 2.0, 2.5, 3.0, 3.5, 4.0

Str(…) converts numbers to text for the ComboBox.

3. Show the form

UserForm1.Show → opens the form on screen.

📌 Logic: clear → fill → show. This macro is a form launcher.

📕 ActivecellShow — Cell Inspector

' Display info about the active cell
Option Explicit Sub ActivecellShow() MsgBox "You are at: " & ActiveCell.Address & _ Chr(13) & "The Formula in the cell is: " & ActiveCell.Formula & Chr(13) & _ "The value of the cell is: " & _ ActiveCell.Value End Sub

🧠 Breakdown

Displays a MsgBox with three pieces of information about the selected cell:

  • ActiveCell.Address → where? e.g. $B$4
  • ActiveCell.Formula → what's written? e.g. =A1+A2
  • ActiveCell.Value → what's the result? e.g. 25

Chr(13) inserts a line break in the message box.

📌 Key distinction:

  • Address = where the cell is
  • Formula = what is written in the cell
  • Value = the result displayed

This macro is a small inspection tool for understanding any cell.

💡 What is Option Explicit?

Option Explicit forces all variables to be declared before use.

If you misspell a variable name, VBA stops and shows an error instead of silently creating a new empty variable. This makes code safer and cleaner.
