üìà PART 2: THE REGRESSION EQUATION (BETA & ALPHA)

Context: We estimate the relationship between a stock and the market to see how the stock historically moves relative to the S&P 500. This is the empirical version of CAPM.

1Ô∏è‚É£ The Statistical Equation
Ri = α + β Rm + ε
  • Ri: Return of the stock
  • Rm: Return of the market
  • α: Intercept (Alpha)
  • β: Beta (Sensitivity to the market)
  • ε: Error term

2Ô∏è‚É£ Beta (β) - Systematic Risk

Measures how sensitive the stock is to market movements. If the market moves 1%, Beta tells you how much the stock moves on average.

β = Cov(Ri , Rm) / Var(Rm)
Excel Implementations:
// Using Covariance: =COVAR(stk ; mkt) / VAR(mkt)
// Using Slope (Equivalent & Simpler): =SLOPE(stock_returns ; market_returns)
  • β > 1 → Aggressive (more volatile)
  • β < 1 → Defensive (less volatile)

3Ô∏è‚É£ Alpha (α) - Abnormal Return

The intercept of the regression. It represents the stock return when the market return is exactly 0. It measures abnormal performance.

In perfect CAPM theory: α should be 0.
Excel Formula:
=INTERCEPT(stock_returns ; market_returns)
  • α > 0 → Stock outperformance
  • α < 0 → Stock underperformance
4Ô∏è‚É£ Important Distinction
1. Regression equation:
Ri = α + β Rm + ε
→ Purely Statistical (based on historical data). Used to estimate Beta empirically.
2. CAPM equation:
E(Ri) = rf + β [ E(Rm) - rf ]
→ Theoretical Equilibrium Model.

üñ•Ô∏è PART 3: OLS REGRESSION — THE MATRIX WAY

The "Engine": Ordinary Least Squares (OLS) chooses α and β that minimize the total squared difference between Actual stock returns and Predicted stock returns (Σ(Ri - Ŷi)²). It finds the "best fitting line".

1Ô∏è‚É£ The Matrix Variables: y = Xβ + ε

Instead of writing one equation per observation, we write everything at once into matrices.

y (Vector)

A single column of Stock Returns (company only).

[ R1 ]
[ R2 ]
[ R3 ]
[ .. ]
X (Matrix) - VERY IMPORTANT

Contains TWO columns of explanatory variables.

  • Col 1: A column of 1s (For the Intercept α)
  • Col 2: Market Returns Rm
[ 1   Rm1 ]
[ 1   Rm2 ]
[ 1   Rm3 ]
[ .   ... ]
Why the column of 1s? Because Ri = α×1 + β×Rm. Without the 1s, the model would force the intercept to exactly 0 (Ri = βRm).

2Ô∏è‚É£ The Coefficient Vector (β)

The order of the result vector strictly follows the order of columns in X:

[ α ]  ← First coefficient (Intercept)
[ β ]  ← Second coefficient (Beta)

3Ô∏è‚É£ The OLS Solving Formula & Excel "Monster"

The mathematical solution that minimizes squared errors to find the best α and β is:

β̂ = (X'X)-1 X'y
To get Intercept and Beta at the same time in Excel, use this exact translation:
=MMULT(
    MINVERSE(
        MMULT(TRANSPOSE(X) ; X)
    ) ;
    MMULT(TRANSPOSE(X) ; y)
)

Output: This returns a 2-cell vertical range (exactly like the β̂ vector). The top cell is Alpha (α), the bottom cell is Beta (β). Matrix formulas are powerful because `SLOPE` and `INTERCEPT` are just simplified versions for one specific variable!

üìâ PART 4: VOLATILITY (σ) & ANNUALIZATION

Concept: Risk is measured by Standard Deviation (σ / Volatility). However, risk is heavily time-dependent. You cannot compare a daily asset with a monthly asset without standardizing the frequency.

1Ô∏è‚É£ Frequency Matters

Using =STDEV(range_of_returns) natively gives you volatility at the frequency of the data (e.g. daily σ for daily data). To compare them properly, we must Annualize them.

2Ô∏è‚É£ The Square Root of Time Rule

Variance (σ²) grows linearly with time (e.g. Var_annual = Var_daily × 250). Since Volatility is the square root of Variance (σ = √Var), Volatility scales with the square root of time!

Data Frequency Annualization Factor Excel Formula
Daily × √250 =STDEV(range) * SQRT(250)
Monthly × √12 =STDEV(range) * SQRT(12)
Weekly × √52 =STDEV(range) * SQRT(52)
‚ö†Ô∏è Trading Days Warning:
We use 250 days for daily data, NOT 365, because financial markets are closed on weekends and holidays!

üéØ FINAL SUMMARY: SIMPLE VS. LOG RETURNS

1. Simple Return
Rt = (Pt / Pt-1) - 1

Standard method for period-by-period calculation. Simple to explain.

2. Log Return
rt = LN(Pt / Pt-1)

Mathematically cleaner for high-level models and complex finance.

üìå Why Use Log Returns (LN)?

Log returns are defined as rt = ln(Pt / Pt-1).

  • • Their main advantage is time additivity: multi-period returns are the sum of individual log returns.
  • • They are fully consistent with continuous compounding and exponential price dynamics.
  • • Most financial models (e.g., Black–Scholes) naturally use log returns.
  • • They allow volatility to scale properly using the square-root-of-time rule (σ × √T).
  • • Log returns are often closer to normality, which simplifies econometric modeling.

For small daily changes, ln(1 + x) ≈ x, so log and simple returns are numerically similar.

Therefore, log returns are preferred in quantitative finance and risk analysis.
