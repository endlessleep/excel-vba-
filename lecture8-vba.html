LECTURE 8: VBA — DATA, LOOPS & FILES

Cross-workbook data transfer, Do/For loop patterns, and file management.

1. Load Data Between Two Workbooks

This macro copies a block of data from a source workbook and pastes it into a destination workbook. VBA always works on the active workbook when using Range, Selection, ActiveSheet, and Paste.

' Load data from one workbook to another
Public Sub LoadData() Workbooks("FichierData.xls").Activate Range("A2:B2").Select Range(Selection, ActiveCell.SpecialCells(xlLastCell)).Select Selection.Copy Workbooks("Lecture8.xls").Activate Range("A16").Select ActiveSheet.Paste End Sub

🧠 Step-by-step breakdown

1. Activate the source workbook

Workbooks("FichierData.xls").Activate

→ all subsequent instructions apply to this workbook.

2. Select and extend the data block

Range("A2:B2").Select Range(Selection, ActiveCell.SpecialCells(xlLastCell)).Select

→ starts at A2:B2, then extends to the last used cell on the sheet.

3. Copy the selected block

Selection.Copy

4. Switch to the destination workbook and paste

Workbooks("Lecture8.xls").Activate Range("A16").Select ActiveSheet.Paste

→ pastes the copied block starting from cell A16.

⚠️ This is recorder-style VBA — it depends on the correct workbook being open, the correct sheet being active, and the last used cell being detected correctly.

📌 Visual logic

source workbook → select data → copy
destination workbook → select target cell → paste

2. Do Loops — Three Variants

Three macros that all use a Do loop to work row by row. They detect how many rows are filled in column A, then write results in columns B, C, and D using three different stopping conditions.

📗 Do Until … Loop

Writes an increasing sequence (100, 101, 102…) in column B.

' Do Until ... Loop
Sub dountill() Dim RCount As Long Dim FRow As Long Dim CCount As Long FRow = 0 RCount = Range("A65000").End(xlUp).Row CCount = Range("A65000").End(xlUp).Column Do Until FRow = RCount Cells(FRow + 1, CCount + 1).Value = 100 + FRow FRow = FRow + 1 Loop End Sub

🧠 Breakdown

RCount = last used row in column A.
CCount = column number of A (= 1).

Cells(FRow + 1, CCount + 1) → writes in column B.

Result: B1 = 100, B2 = 101, B3 = 102…

Stopping condition: Do Until FRow = RCount
→ repeat until the counter reaches the last row.

📕 Do While … Loop

Writes a constant value (100) in column C.

' Do While ... Loop
Sub dowhile() Dim RCount As Long Dim FRow As Long Dim CCount As Long FRow = 0 RCount = Range("A65000").End(xlUp).Row CCount = Range("A65000").End(xlUp).Column Do While FRow <> RCount Cells(FRow + 1, CCount + 2).Value = 100 FRow = FRow + 1 Loop End Sub

🧠 Breakdown

CCount + 2 = 3 → writes in column C.

Result: C1 = 100, C2 = 100, C3 = 100…

Stopping condition: Do While FRow <> RCount
→ repeat while the counter is different from the last row.

Logically equivalent to Do Until FRow = RCount — both stop at the same moment.

📙 Do … Loop Until

Sums columns B + C and writes the result in column D.

' Do ... Loop Until
Sub dountill2() Dim RCount As Long Dim FRow As Long Dim CCount As Long FRow = 0 RCount = Range("A65000").End(xlUp).Row CCount = Range("A65000").End(xlUp).Column Do Cells(FRow + 1, CCount + 3).Value = Cells(FRow + 1, CCount + 2).Value + Cells(FRow + 1, CCount + 1).Value FRow = FRow + 1 Loop Until FRow = RCount End Sub

🧠 Breakdown

CCount + 3 = 4 → writes in column D.

D = C + B. Example: if B1 = 100 and C1 = 100, then D1 = 200.

Stopping condition at the end: Loop Until FRow = RCount
→ runs the block first, then checks. Different structure from the first two.

💡 Shared logic

All three macros use Range("A65000").End(xlUp).Row to find the last used row in column A. Column A is only used as a reference — results are written in B, C, D.

📋 Three loop styles

  • Do Until … Loop — condition at the top
  • Do While … Loop — condition at the top (opposite logic)
  • Do … Loop Until — condition at the bottom

3. Opening a Workbook

Three procedures that manage file-based data import: one opens from a typed filename, one clears imported data, and one uses a file picker. The actual copy/paste logic is delegated to LoadData.

📗 Button 1 — Open from typed filename

' Open file from typed filename
Private Sub CommandButton1_Click() Dim DataFile As Object Dim DataWB As Excel.Workbook Set DataFile = CreateObject("Scripting.FileSystemObject") Filename = InputBox("File Name", "Input", CurDir & "\Lecture8_Data.xls") If DataFile.FileExists(Filename) = True Then Workbooks.Open(Filename) Set DataWB = ActiveWorkbook LoadData DataWB.Close Else Application.DisplayAlerts = False Application.ScreenUpdating = False MsgBox("Specified file does not exist!") End If End Sub

🧠 Breakdown

CreateObject("Scripting.FileSystemObject") → creates an object that can check files on disk.

DataFile.FileExists(Filename) → verifies whether the file path is valid.

If the file exists: open → run LoadData → close.

The file is only opened temporarily as a source, then closed after import.

📕 Button 2 — Clear imported data

' Clear imported data starting from A16
Private Sub CommandButton2_Click() Range("A16").Activate If ActiveCell <> "" Then Range("A16:B16").Select Range(Selection, ActiveCell.SpecialCells(xlLastCell)).Select Selection.ClearContents Range("A16").Select End If End Sub

🧠 Breakdown

Checks if A16 is not empty → extends the selection to the last used cell → clears values only (not formatting).

Then moves back to A16.

📙 OpenFile — File Chooser

' Open file with file picker dialog
Sub OpenFile() nsName1 = ActiveWorkbook.Name MsgBox("Select file to open") NewFN = Application.GetOpenFilename(fileFilter:="Excel Files (.xls), .xls", Title:="SELECT FILE HERE") If NewFN = False Then MsgBox "Stopping because you did not select a file" Exit Sub Else Set SampleFile = Application.Workbooks.Open(Filename:=NewFN) End If nsName0 = SampleFile.Name Workbooks(nsName0).Sheets("Data").Select LoadData End Sub

🧠 Breakdown

Application.GetOpenFilename(…) → opens the standard file picker window.

If the user cancels → the macro stops with Exit Sub.

If a file is selected: open → go to "Data" sheet → run LoadData.

More user-friendly than the InputBox version — no path typing needed.

📌 Note: Unlike Button 1, this macro does not close the source workbook afterwards.

💡 Architecture

These procedures separate the workflow into two parts:

  • File handling — how the file is chosen, opened, closed
  • Data loading — delegated to LoadData

4. For Loop — Five Patterns

Five examples showing how a For…Next loop can be adapted: basic looping, looping through all rows, skipping rows with Step, looping backwards, and stopping early with Exit For.

1️⃣ Basic loop

' Basic For loop — writes 101…110 in column A
Sub forexample() Dim i As Integer Dim c As Integer c = 1 For i = 1 To 10 Cells(i, c).Value = 100 + Cells(i, c).Row Next i End Sub

🧠 Breakdown

Writes in column A (c = 1), rows 1 to 10.

Each cell = 100 + row number → A1 = 101, A2 = 102 … A10 = 110.

This is the most basic form of For…Next.

2️⃣ Step 1 (explicit default)

' Step 1 — writes A + i in column B
Sub forexample2() LastRow = Range("A65000").End(xlUp).Row FirstCol = Range("A65000").End(xlUp).Column For i = 1 To LastRow Step 1 Cells(i, FirstCol + 1).Value = Cells(i, FirstCol) + i Next i End Sub

🧠 Breakdown

Writes in column B (FirstCol + 1 = 2).

Each cell = value in column A + row number.

If A1 = 10, A2 = 20, A3 = 30 → B1 = 11, B2 = 22, B3 = 33.

Step 1 is the default — shown here for complete syntax.

3️⃣ Step 2 (skip rows)

' Step 2 — processes every other row in column C
Sub forexample3() LastRow = Range("A65000").End(xlUp).Row FirstCol = Range("A65000").End(xlUp).Column For i = 1 To LastRow Step 2 Cells(i, FirstCol + 2).Value = Cells(i, FirstCol + 1) + i Next i End Sub

🧠 Breakdown

Writes in column C (FirstCol + 2 = 3).

Processes row 1, 3, 5, 7… — skips every second row.

Each cell = value in column B + row number.

4️⃣ Step 3 (wider skip)

' Step 3 — processes every third row in column D
Sub forexample4() LastRow = Range("A65000").End(xlUp).Row FirstCol = Range("A65000").End(xlUp).Column For i = 1 To LastRow Step 3 Cells(i, FirstCol + 3).Value = Cells(i, FirstCol) + i Next i End Sub

🧠 Breakdown

Writes in column D (FirstCol + 3 = 4).

Processes row 1, 4, 7, 10… — every third row only.

Each cell = value in column A + row number.

5️⃣ Step -1 & Exit For (backward + early stop)

' Step -1 — backwards loop with immediate exit
Sub forexample5() LastRow = Range("A65000").End(xlUp).Row FirstCol = Range("A65000").End(xlUp).Column For i = LastRow To 1 Step -1 Cells(i, FirstCol + 4).Value = Cells(i, FirstCol) + i Exit For Next i MsgBox "First Value Found" End Sub

🧠 Breakdown

Writes in column E (FirstCol + 4 = 5).

The loop starts at LastRow and goes backwards to row 1 with Step -1.

But Exit For stops the loop immediately after the first iteration.

So in practice, only one value is written — on the last used row, in column E.

This demonstrates two ideas: backwards looping and early exit.

💡 Key insight

A For…Next loop can move forward, skip rows, move backward, or stop after the first match. Same structure, many behaviors.
