1. Variance-Covariance Matrix

This function takes a table of asset returns (each column = one asset) and computes the full variance-covariance matrix. It compares every column with every other column using Excel's built-in COVAR function.

' Variance-Covariance Function
Option Explicit Function VarCovar(rng As Range) As Variant Dim i As Integer Dim j As Integer Dim numCols As Integer numCols = rng.Columns.Count Dim matrix() As Double ReDim matrix(numCols - 1, numCols - 1) For i = 1 To numCols For j = 1 To numCols matrix(i - 1, j - 1) = Application.WorksheetFunction.Covar(rng.Columns(i), rng.Columns(j)) Next j Next i VarCovar = matrix End Function

🧠 Step-by-step breakdown

1. Identify the problem

We want a matrix where each cell = Cov(asset i, asset j). So we must compare every column with every other column.

2. Count the number of assets

numCols = rng.Columns.Count

→ determines how big the output matrix should be (numCols × numCols).

3. Create the matrix (dynamic storage)

Dim matrix() As Double ReDim matrix(numCols - 1, numCols - 1)

→ ReDim creates a dynamic array whose size depends on the input data.

4. Core logic — the double loop

For i = 1 To numCols For j = 1 To numCols

→ generates all combinations of (asset i, asset j). Without this double loop you cannot build the full matrix.

5. Key computation

Application.WorksheetFunction.Covar(rng.Columns(i), rng.Columns(j))

→ Application.WorksheetFunction allows VBA to use any Excel formula — same as writing =COVAR(…) in a cell.

6. Store & return

matrix(i-1, j-1) = … fills the matrix cell by cell.

VarCovar = matrix returns the full matrix as an array output.

2. Real NPV (Date-Based Discounting)

This function computes a Net Present Value using real dates instead of fixed periods. Each cash flow is discounted based on the actual time elapsed between dates.

' Real NPV Function (date-based)
Option Explicit Function RealNPV(varDiscountRate As Double, varDate As Range, varCF As Range) As Double Dim i As Integer Dim numrows As Integer Dim dt As Date Dim t As Double Dim dr As Double numrows = varCF.Rows.Count dt = varDate.Cells(1, 1) dr = 1 For i = 1 To numrows t = DateDiff("d", dt, varDate.Cells(i, 1)) / 365 RealNPV = RealNPV + (varCF.Cells(i, 1) / (dr * (1 + varDiscountRate * t) ^ t)) dr = dr * (1 + varDiscountRate * t) ^ t dt = varDate.Cells(i, 1) Next i End Function

🧠 Step-by-step breakdown

Key idea

We do not assume fixed periods (1 year, 2 years…). Instead, we compute the exact time between each cash flow using real dates.

1. Time calculation

t = DateDiff("d", dt, varDate.Cells(i, 1)) / 365

→ gives time in years between two consecutive cash flows (number of days ÷ 365).

2. Discounting logic

RealNPV = RealNPV + CFᵢ / (dr × (1 + r × tᵢ)^tᵢ)

→ each cash flow is discounted using a cumulative discount factor dr.

3. Cumulative discount factor

dr = dr * (1 + varDiscountRate * t) ^ t

→ dr keeps track of all past discounting. It grows over time, reducing the present value of future cash flows.

4. Why dt is updated

dt = varDate.Cells(i, 1)

→ dt stores the previous date. This ensures we compute tᵢ = Dateᵢ − Dateᵢ₋₁ (the gap between consecutive dates, not from the first date).

💡 Key insight: Cash flows are discounted step by step with irregular time intervals. The function builds the total discount progressively.

3. Black-Scholes Option Pricing

📘 What is Black-Scholes?

An option is a right (not an obligation) to buy or sell an asset at a fixed price in the future.

  • Call option = the right to buy later at a pre-agreed price.
  • Put option = the right to sell later at a pre-agreed price.

The question Black-Scholes answers: "How much is this right worth today?"

The model estimates the expected future gain and discounts it back to today. The general formula structure is:

Option Value = Adjusted Stock Component − Adjusted Strike Component

The price depends on 6 inputs:

  • S = current stock price
  • X = exercise (strike) price
  • r = risk-free interest rate
  • q = continuous dividend yield
  • tyr = time to maturity (in years)
  • sigma (σ) = volatility

⚙️ Module-Level Declarations

Option Explicit Option Base 1

Option Explicit forces you to declare all variables (avoids hidden mistakes). Option Base 1 sets default array lower bound to 1.

📗 BSCallValue — Call Option Price

📐 Formula: c = S·e−qT·N(d₁) − X·e−rT·N(d₂)
' Black-Scholes CALL value
Function BSCallValue(S, X, r, q, tyr, sigma) As Double Dim ert, eqt Dim DOne, DTwo, NDOne, NDTwo ert = Exp(-r * tyr) eqt = Exp(-q * tyr) DOne = (Log(S / X) + (r - q + 0.5 * sigma ^ 2) * tyr) / (sigma * Sqr(tyr)) DTwo = (Log(S / X) + (r - q - 0.5 * sigma ^ 2) * tyr) / (sigma * Sqr(tyr)) NDOne = Application.NormSDist(DOne) NDTwo = Application.NormSDist(DTwo) BSCallValue = S * eqt * NDOne - X * ert * NDTwo End Function

🧠 Line-by-line breakdown

ert = Exp(-r * tyr)

Discount factor for the strike price: e−rT. Paying X in the future is worth less today.

eqt = Exp(-q * tyr)

Adjusts the stock price for dividends: e−qT. If the stock pays dividends, its expected future value is slightly reduced.

DOne = … (d₁)

A score that summarizes how favorable the situation is: ratio S/X, interest rate, dividend yield, volatility, and time. A larger d₁ means the option is more "in the money".

DTwo = … (d₂)

Very close to d₁ but slightly smaller: d₂ = d₁ − σ√T.

NDOne / NDTwo

Application.NormSDist() converts d₁ and d₂ into cumulative normal probabilities N(d₁) and N(d₂) — think of them as probability-like weights.

Final line

BSCallValue = S × eqt × N(d₁) − X × ert × N(d₂)

= expected benefit (adjusted stock) − expected cost (adjusted strike).

📕 BSPutValue — Put Option Price

📐 Formula: p = X·e−rT·N(−d₂) − S·e−qT·N(−d₁)
' Black-Scholes PUT value
Function BSPutValue(S, X, r, q, tyr, sigma) As Double Dim ert, eqt Dim DOne, DTwo, NDOne, NDTwo ert = Exp(-r * tyr) eqt = Exp(-q * tyr) DOne = (Log(S / X) + (r - q + 0.5 * sigma ^ 2) * tyr) / (sigma * Sqr(tyr)) DTwo = (Log(S / X) + (r - q - 0.5 * sigma ^ 2) * tyr) / (sigma * Sqr(tyr)) NDOne = Application.NormSDist(-DOne) NDTwo = Application.NormSDist(-DTwo) BSPutValue = X * ert * NDTwo - S * eqt * NDOne End Function

🧠 What changes vs. the Call?

The structure is almost identical to BSCallValue. Same inputs, same discount factors, same d₁ and d₂.

Key difference #1 — Negative d values:

NDOne = Application.NormSDist(-DOne) NDTwo = Application.NormSDist(-DTwo)

For the put we use N(−d₁) and N(−d₂). This reflects the opposite logic: a call benefits when the stock goes up, a put benefits when the stock goes down.

Key difference #2 — Reversed formula:

BSPutValue = X × ert × N(−d₂) − S × eqt × N(−d₁)

Compared with the call, the stock and strike components are swapped.

🔀 BSOptValue — General Option Pricing

🔑 Key variable: iopt = 1 → Call  |  iopt = -1 → Put
' General Black-Scholes (Call OR Put)
Function BSOptValue(iopt, S, X, r, q, tyr, sigma) Dim ert, eqt, NDOne, NDTwo ert = Exp(-r * tyr) eqt = Exp(-q * tyr) NDOne = Application.NormSDist(iopt * BSDOne(S, X, r, q, tyr, sigma)) NDTwo = Application.NormSDist(iopt * BSDTwo(S, X, r, q, tyr, sigma)) BSOptValue = iopt * (S * eqt * NDOne - X * ert * NDTwo) End Function

🧠 The iopt trick

Instead of writing two separate formulas, this function uses one general expression and changes the sign via iopt:

If iopt = 1 (Call):

N(iopt × d₁) = N(d₁) and N(iopt × d₂) = N(d₂) → call formula.

If iopt = -1 (Put):

N(iopt × d₁) = N(−d₁) and N(iopt × d₂) = N(−d₂) → put formula.

Final line: iopt × (S·eqt·N − X·ert·N) flips the sign for puts automatically.

✅ This is more compact, professional, and easier to maintain — one function handles both option types.

🔧 Helper — BSDOne (d₁)

Function BSDOne(S, X, r, q, tyr, sigma) BSDOne = (Log(S / X) + (r - q + 0.5 * sigma ^ 2) * tyr) / (sigma * Sqr(tyr)) End Function

d₁ is an intermediate score combining S/X ratio, rates, volatility, and time. A larger d₁ = option more "in the money".

🔧 Helper — BSDTwo (d₂)

Function BSDTwo(S, X, r, q, tyr, sigma) BSDTwo = (Log(S / X) + (r - q - 0.5 * sigma ^ 2) * tyr) / (sigma * Sqr(tyr)) End Function

d₂ = d₁ − σ√T. BS uses N(d₁) for the stock part and N(d₂) for the strike part — these are the two key building blocks.
