LECTURE 7: VBA — SUM, PRODUCT & RANDOM

Function vs Sub patterns — building core operations from scratch.

1. SUM

Two ways to sum a range of cells: a Function that receives a range directly, and a Sub that navigates through the worksheet cell by cell.

📗 SumaF — Function

' Sum — Function approach
Public Function SumaF(numbers As Range) As Double Dim i As Integer SumaF = numbers(1, 1).Value For i = 2 To numbers.Count SumaF = SumaF + numbers(i, 1).Value Next i End Function

🧠 Step-by-step breakdown

This function receives a range of cells called numbers.

1. Initialize with the first value

SumaF = numbers(1, 1).Value

2. Loop through the rest

For i = 2 To numbers.Count

→ starts at 2 because the first cell was already used to initialize the sum.

3. Accumulate

SumaF = SumaF + numbers(i, 1).Value

→ adds each cell to the running total.

📌 Key detail

numbers(1,1) means the first cell of the given range. If the range is B3:B6, then numbers(1,1) = B3, numbers(2,1) = B4, etc.

📕 TheSum — Sub

' Sum — Sub approach
Public Sub TheSum() Dim varSum As Double Range("b3").Activate varSum = ActiveCell.Value Do ActiveCell.Offset(1, 0).Activate If ActiveCell.Value <> "" And VBA.IsNumeric(ActiveCell.Value) = True Then varSum = varSum + ActiveCell.Value End If Loop Until ActiveCell.Text = "" ActiveCell.Value = varSum ActiveCell.Font.Bold = True End Sub

🧠 Step-by-step breakdown

This macro starts in cell B3 and stores its value in varSum.

1. Move down one cell at a time

ActiveCell.Offset(1, 0).Activate

→ Offset(1, 0) = move 1 row down, same column.

2. Check & accumulate

If the new cell contains a number, add it to the total.

3. Stop condition

Loop Until ActiveCell.Text = ""

→ the empty cell is used as the stopping signal.

4. Write result

The macro writes the total into the empty cell and formats it in bold.

📌 Example

If B3 = 5, B4 = 7, B5 = 2, and B6 is empty → computes 5 + 7 + 2 = 14, writes 14 into B6 in bold.

💡 Function vs Sub

Function = better when the range is already known.

Sub = better when the macro must move through Excel and decide where to stop.

⚠️ Button use

A button can directly run a Sub, but not usually a Function.

  • A Sub performs an action in Excel
  • A Function returns a value

→ TheSum can be assigned directly to a button.
→ SumaF should usually be called inside a Sub.

2. PRODUCT

Same pattern as SUM, but with multiplication instead of addition. Again, two approaches: Function and Sub.

📗 ProductF — Function

' Product — Function approach
Public Function ProductF(numbers As Range) As Double Dim i As Integer ProductF = numbers(1, 1).Value For i = 2 To numbers.Count ProductF = ProductF * numbers(i, 1).Value Next i End Function

🧠 Step-by-step breakdown

This function receives a range of cells called numbers.

1. Initialize with the first value

ProductF = numbers(1, 1).Value

2. Loop through the rest

For i = 2 To numbers.Count

→ starts at 2 because the first cell was already used to initialize the product.

3. Accumulate

ProductF = ProductF * numbers(i, 1).Value

→ multiplies each cell into the running product.

📌 Key detail

If the range is C3:C6, then numbers(1,1) = C3, numbers(2,1) = C4, etc.

📕 TheProduct — Sub

' Product — Sub approach
Public Sub TheProduct() Dim varProduct As Double Range("c3").Activate varProduct = ActiveCell.Value Do ActiveCell.Offset(1, 0).Activate If ActiveCell.Value <> "" And VBA.IsNumeric(ActiveCell.Value) = True Then varProduct = varProduct * ActiveCell.Value End If Loop Until ActiveCell.Text = "" ActiveCell.Value = varProduct ActiveCell.Font.Bold = True End Sub

🧠 Step-by-step breakdown

This macro starts in cell C3 and stores its value in varProduct.

1. Move down one cell at a time

ActiveCell.Offset(1, 0).Activate

→ Offset(1, 0) = move 1 row down, same column.

2. Check & accumulate

If the new cell contains a number, multiply it into the running product.

3. Stop & write

When the empty cell is reached, the macro writes the product and formats it in bold.

📌 Example

If C3 = 2, C4 = 3, C5 = 4, and C6 is empty → computes 2 × 3 × 4 = 24, writes 24 into C6 in bold.

⚠️ Button use

→ TheProduct can be assigned directly to a button.
→ ProductF should usually be called inside a Sub.

3. RANDOM

Two ways to fill a range with random numbers: one using the current Selection, the other prompting the user via an InputBox.

📗 MakroRn — Selection-Based

' Random — Selection-based
Sub MakroRn() 'Ctrl+n rw = Selection.Rows.Count cl = Selection.Columns.Count For i = 1 To rw For j = 1 To cl Selection.Cells(i, j).Value = Rnd Next j Next i End Sub

🧠 Step-by-step breakdown

This macro works on the cells currently selected in Excel.

1. Count the selection dimensions

rw = Selection.Rows.Count cl = Selection.Columns.Count

2. Double loop to fill every cell

→ the first loop goes through rows, the second through columns.

3. Write a random number

Selection.Cells(i, j).Value = Rnd

→ every cell receives a random number between 0 and 1.

📌 Example

If the selected range is A1:B2, the macro fills A1, B1, A2, B2 with random numbers.

📕 RandomSub — InputBox-Based

' Random — InputBox-based
Sub RandomSub() pos = InputBox("Input array specification e.g. a1:b3") rw = Range(pos).Rows.Count cl = Range(pos).Columns.Count For i = 1 To rw For j = 1 To cl Range(pos).Cells(i, j).Value = Rnd Next j Next i End Sub

🧠 Step-by-step breakdown

Same filling logic, but instead of using the current selection, it asks the user to type a range.

1. Prompt the user

pos = InputBox("Input array specification e.g. a1:b3")

→ pos stores the range address as text (e.g. "A1:B3").

2. Convert text → range object

Range(pos)

→ turns the text into a real Excel range.

3. Fill with random numbers

The two loops work exactly like MakroRn.

📌 Key difference

The only difference is how the target area is chosen:
→ Selection in MakroRn
→ InputBox + Range(pos) in RandomSub

⚠️ Button use

Both MakroRn and RandomSub are Sub procedures → both can be assigned directly to a button.

  • MakroRn needs the user to select a range first
  • RandomSub asks for the range through an input box

💡 Core pattern

Both methods share the same filling logic:

  • Count rows
  • Count columns
  • Loop through the whole area
  • Place Rnd in each cell
