LECTURE 10: WORD AUTOMATION FROM EXCEL

Using Excel VBA to open, edit, and save a Word document automatically.

1. Word Automation — OpenWordDoc

This macro uses Excel VBA to control Word: it opens a document, modifies paragraphs depending on their font color, copies an Excel range into the document, saves it under a new name, and closes Word. Excel drives Word automatically.

' Word Automation from Excel
Sub OpenWordDoc() Dim objWord Dim objDoc Dim tbl As Excel.Range Dim WordTable As Word.Table Set objWord = CreateObject("Word.Application") Set tbl = Worksheets("data").Range("G18:H19") Set objDoc = objWord.Documents.Open("PATH\original_document.docx") objWord.Visible = True objDoc.Label1.Caption = objDoc.Label1.Caption & " Test" For Each oPara In objDoc.Content.Paragraphs If oPara.Range.Font.Color = wdColorRed And Worksheets("data").CheckBox1.Value = True Then If oPara.Range.Font.Color = wdColorRed Then oPara.Range.Font.Color = wdColorBlack Else If oPara.Range.Font.Color = wdColorRed Then oPara.Range.Delete End If If oPara.Range.Font.Color = wdColorGreen Then oPara.Range.Font.Color = wdColorBlue Next oPara tbl.Copy objDoc.Content.Select objWord.Selection.Collapse wdCollapseEnd objWord.Selection.PasteExcelTable LinkedToExcel:=False, WordFormatting:=Yes, RTF:=False objDoc.SaveAs Filename:="PATH\new_document.docx" objWord.Quit EndRoutine: Application.ScreenUpdating = True Application.EnableEvents = True Application.CutCopyMode = False End Sub

📗 Step 1 — Setup & Open Word

Set objWord = CreateObject("Word.Application") Set tbl = Worksheets("data").Range("G18:H19") Set objDoc = objWord.Documents.Open("PATH\original_document.docx") objWord.Visible = True

🧠 Breakdown

CreateObject("Word.Application")

→ the key instruction — creates a Word application from inside Excel. This is what enables cross-application automation.

tbl = Worksheets("data").Range("G18:H19")

→ selects the Excel range that will later be pasted into Word.

objWord.Documents.Open(…)

→ opens an existing Word document for modification.

objWord.Visible = True

→ makes Word visible on screen (otherwise it runs in the background).

📕 Step 2 — Paragraph Formatting Logic

For Each oPara In objDoc.Content.Paragraphs If oPara.Range.Font.Color = wdColorRed And Worksheets("data").CheckBox1.Value = True Then If oPara.Range.Font.Color = wdColorRed Then oPara.Range.Font.Color = wdColorBlack Else If oPara.Range.Font.Color = wdColorRed Then oPara.Range.Delete End If If oPara.Range.Font.Color = wdColorGreen Then oPara.Range.Font.Color = wdColorBlue Next oPara

🧠 Breakdown

The loop processes the Word document paragraph by paragraph.

🔴 Red paragraphs — two outcomes:

  • CheckBox1 checked → red becomes black (kept but recolored)
  • CheckBox1 unchecked → red paragraph is deleted

🟢 Green paragraphs:

→ always become blue.

💡 The Excel worksheet acts as a control panel for the Word document. The checkbox value influences how paragraphs are treated — this is cross-application automation.

📙 Step 3 — Paste Excel Table into Word

tbl.Copy objDoc.Content.Select objWord.Selection.Collapse wdCollapseEnd objWord.Selection.PasteExcelTable LinkedToExcel:=False, WordFormatting:=Yes, RTF:=False

🧠 Breakdown

1. tbl.Copy → copies the Excel range (G18:H19).

2. objDoc.Content.Select → selects the entire Word document content.

3. Selection.Collapse wdCollapseEnd → moves the insertion point to the end of the document.

4. PasteExcelTable → inserts the Excel range as a table in Word.

  • LinkedToExcel:=False → the Word table becomes independent (no link back to Excel)
  • WordFormatting:=Yes → uses Word's formatting style
  • RTF:=False → no rich text formatting

📘 Step 4 — Save & Close

objDoc.SaveAs Filename:="PATH\new_document.docx" objWord.Quit EndRoutine: Application.ScreenUpdating = True Application.EnableEvents = True Application.CutCopyMode = False

🧠 Breakdown

SaveAs → saves the modified document under a new name. The original file is not overwritten.

objWord.Quit → closes Word entirely.

Cleanup:

  • ScreenUpdating = True → turns screen refresh back on
  • EnableEvents = True → re-enables Excel events
  • CutCopyMode = False → clears the clipboard

This is standard cleanup after automation tasks.

⚠️ Important note

The line objDoc.Label1.Caption = … is not standard for a normal Word document. A classic .docx does not contain Label1.Caption. This may only work if the document contains a custom ActiveX control — in a normal document, this line will fail.

📋 Summary — Automation Flow

Step Action Key instruction
1 Create Word application CreateObject("Word.Application")
2 Open document Documents.Open(…)
3 Edit paragraphs by color For Each oPara …
4 Copy Excel range tbl.Copy
5 Paste into Word PasteExcelTable
6 Save as new file SaveAs
7 Close Word objWord.Quit

💡 Key insight

Excel is the controller, Word is being controlled.

Excel provides data (the table) and controls (the checkbox) — Word receives the edits and the pasted content.

This is cross-application automation: one macro in Excel drives the entire editing process in Word.
